# Гайд по контрибьютингу

Этот гайд описывает, как добавлять новые центры и аккуратно обновлять существующую информацию в `tatarverse.cc`.

## Общие правила

- Делайте один pull request на одну тему: новый центр, правка данных, перевод или небольшое исправление.
- Не форматируйте и не меняйте соседние файлы без необходимости.
- Для фактических правок добавляйте источник: сайт, соцсеть, официальный пост, страницу организации или другую проверяемую ссылку.
- Если данных нет, не пишите заглушки вроде `неизвестно`, `n/a`, `-`. Лучше пропустить поле.
- Сохраняйте нейтральный тон: без рекламы, оценок и неподтвержденных формулировок.
- Не меняйте `tbk-*` имена файлов: они работают как публичные стабильные slug.

## Где лежат центры

Основные русские карточки центров лежат в:

```txt
src/data/centers_formatted/
```

Английские переводы центров лежат в:

```txt
src/data/centers_i18n/en/
```

Имена файлов должны совпадать между версиями:

```txt
src/data/centers_formatted/tbk-366.mdx
src/data/centers_i18n/en/tbk-366.mdx
```

Русская версия является исходной. Если меняете исходную карточку, проверьте, нужна ли такая же фактическая правка в английском переводе.

## Как добавить новый центр

1. Найдите следующий свободный `tbk-*` номер в `src/data/centers_formatted/`.
2. Создайте новый `.mdx` файл с этим номером.
3. Заполните frontmatter по схеме из `src/content.config.ts`.
4. Добавьте краткий MDX-контент ниже frontmatter: описание, ссылки, особенности, если они подтверждены источниками.
5. По возможности добавьте английскую версию в `src/data/centers_i18n/en/` с тем же именем файла.

Минимальный пример:

```mdx
---
title: Название центра
type: Зарубежный
category: Татарский
source: https://example.com/
summary: Краткое нейтральное описание центра.
location:
  country: Казахстан
  city: Алматы
---

# Название центра

Краткое описание с проверяемой информацией.

## Ссылки

- [Официальный сайт](https://example.com/)
```

## Как менять существующую информацию

- Меняйте только поля и текст, которые действительно устарели или ошибочны.
- Не переименовывайте `tbk-*` файл.
- Не удаляйте старую информацию без причины; если данные устарели, замените их на подтвержденные.
- При изменении `title`, `summary`, `location`, `source`, `type` или `category` проверьте, что источник подтверждает правку.
- Сохраняйте одинаковый фактический смысл между исходными и переведенными файлами.

## Схема полей

Схема центров находится в `src/content.config.ts`. Она строгая: лишние поля в frontmatter не проходят проверку.

Разрешенные поля:

- `title` - обязательная строка.
- `pubDate` - необязательная строка с датой.
- `type` - необязательный тип центра.
- `category` - необязательная категория центра.
- `source` - необязательный URL источника.
- `summary` - необязательное краткое описание.
- `location` - необязательный объект локации.

Поля `location`:

- `country`
- `city`
- `region`
- `flag`

## Категории и типы

В проекте нет отдельного поля `tags` для центров. Для группировки используются поля `category` и `type`, заданные через Zod enum.

Разрешенные `category`:

- `Татарский`
- `Татаро-Башкирский`
- `Башкирский`
- `Крымотатарский`

Разрешенные `type`:

- `Регион РФ`
- `Зарубежный`
- `Онлайн`

Не добавляйте новые категории или типы прямо в MDX. Если нужна новая категория, сначала обновляется схема и логика интерфейса.

## Переводы интерфейса

Строки интерфейса лежат в `src/i18n/locales/ru.ts` и `src/i18n/locales/en.ts`.

Если добавляете UI-текст, добавьте одинаковый ключ в оба файла и используйте `useTranslations(locale)` в Astro-компонентах. Не оставляйте видимый пользователю текст захардкоженным.

## Проверка

Для обычной правки контента достаточно внимательно проверить измененные `.mdx` файлы. Если менялась схема, маршруты, локализация или общая логика данных, используйте точечную проверку:

```bash
bunx astro sync
```

Полную сборку запускайте только когда изменение затрагивает маршруты, Astro config, интеграции или общую логику данных сайта.
