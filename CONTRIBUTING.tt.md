# Кертем ясау кулланмасы

Бу кулланма tatarverse.cc проектына яңа үзәкләр өстәү һәм булган мәгълүматны дөрес яңарту тәртибен аңлата.

## Гомуми кагыйдәләр

- Бер pull request эчендә бер генә тема калдырыгыз: яңа үзәк, мәгълүмат төзәтү, тәрҗемә яки кечкенә хата.
- Кирәк булмаса, күрше файлларны форматламагыз һәм үзгәртмәгез.
- Фактларны үзгәрткәндә чыганак күрсәтегез: сайт, социаль челтәр, рәсми язма, оешма битe яки башка тикшерелә торган сылтама.
- Мәгълүмат билгесез булса, кырны төшереп калдырыгыз. `билгесез`, `n/a`, `-` кебек тутыргычлар язмагыз.
- Тон нейтраль булсын: реклама, бәяләү һәм расланмаган сүзләр кулланмагыз.

## Үзәкләр кайда саклана

Төп үзәк файллары монда:

```txt
src/data/centers_formatted/
```

Тәрҗемә файллары монда:

```txt
src/data/centers_i18n/en/
src/data/centers_i18n/tt/
src/data/centers_i18n/qt/
```

Рус версиясе төп чыганак булып тора. Файл исемнәре телләр арасында туры килергә тиеш, мәсәлән:

```txt
src/data/centers_formatted/tbk-366.mdx
src/data/centers_i18n/en/tbk-366.mdx
src/data/centers_i18n/tt/tbk-366.mdx
src/data/centers_i18n/qt/tbk-366.mdx
```

## Яңа үзәк өстәү

1. `src/data/centers_formatted/` эчендә чираттагы буш `tbk-*` номерын табыгыз.
2. Шул номер белән яңа `.mdx` файл булдырыгыз.
3. Frontmatter өлешен `src/content.config.ts` эчендәге Zod схемасы буенча тутырыгыз.
4. Frontmatter астына кыска MDX эчтәлек өстәгез: тасвирлама, сылтамалар һәм чыганак белән расланган үзенчәлекләр.
5. Мөмкин булса, шул ук файл исемe белән `src/data/centers_i18n/en`, `tt`, `qt` версияләрен дә өстәгез.

Минималь үрнәк:

```mdx
---
title: Үзәк исеме
type: Зарубежный
category: Татарский
source: https://example.com/
summary: Үзәк турында кыска нейтраль тасвирлама.
location:
  country: Казахстан
  city: Алматы
---

# Үзәк исеме

Тикшерелә торган мәгълүматка нигезләнгән кыска тасвирлама.

## Сылтамалар

- [Рәсми сайт](https://example.com/)
```

## Булган мәгълүматны яңарту

- Чыннан да искергән яки хаталы кырларны һәм текстны гына үзгәртегез.
- `tbk-*` файл исемен үзгәртмәгез: ул тотрыклы slug булып кулланыла.
- Сәбәпсез иске мәгълүматны бетермәгез; искергән мәгълүматны расланган мәгълүмат белән алыштырыгыз.
- `title`, `summary`, `location`, `source`, `type` яки `category` үзгәрсә, чыганак моны расларга тиеш.
- Рус версиясе үзгәрсә, тәрҗемә версияләрен дә яңартырга кирәкме икәнен тикшерегез.

## Zod кыр схемасы

Схема `src/content.config.ts` эчендә билгеләнгән. Ул катгый: frontmatter эчендә артык кырлар рөхсәт ителми.

Рөхсәт ителгән кырлар:

- `title` - мәҗбүри юл.
- `pubDate` - мәҗбүри булмаган дата юлы.
- `type` - мәҗбүри булмаган үзәк төре.
- `category` - мәҗбүри булмаган үзәк категориясе.
- `source` - мәҗбүри булмаган чыганак URL-ы.
- `summary` - мәҗбүри булмаган кыска тасвирлама.
- `location` - мәҗбүри булмаган урын объекты.

`location` кырлары:

- `country`
- `city`
- `region`
- `flag`

## Категорияләр һәм төрләр

Үзәкләр өчен аерым `tags` кыры хәзер юк. Төркемләү `category` һәм `type` кырлары аша эшли; алар Zod enum белән чикләнгән.

Рөхсәт ителгән `category` кыйммәтләре:

- `Татарский`
- `Татаро-Башкирский`
- `Башкирский`
- `Крымотатарский`

Рөхсәт ителгән `type` кыйммәтләре:

- `Регион РФ`
- `Зарубежный`
- `Онлайн`

Яңа категория яки төрне турыдан-туры MDX эченә өстәмәгез. Яңа кыйммәт кирәк булса, башта Zod схемасын һәм бәйле UI логикасын яңартырга кирәк.

## Тикшерү

Гади контент төзәтүләре өчен үзгәртелгән `.mdx` файлларны игътибар белән карау җитә. Схема, маршрутлар яки уртак мәгълүмат логикасы үзгәрсә, көчлерәк проект тикшерүе кирәк.
