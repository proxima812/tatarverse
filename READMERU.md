# tatarverse.cc

Статический мультиязычный сайт на Astro о татарских и башкирских сообществах, культурных центрах, источниках, переводах и справочных материалах.

Английская документация находится в [README.md](./README.md).

## Статус проекта

Проект готовится к open source. Самые полезные вклады: проверенные обновления данных о центрах, переводы, источники, исправления доступности и небольшие UI-улучшения без изменения общего визуального языка.

## Стек

- Astro 6 со статической сборкой
- MDX content collections
- Tailwind CSS v4 через `@tailwindcss/vite`
- Bun как предпочтительный package manager
- Мультиязычные маршруты: `ru`, `en`, `tt`, `qt`, `uk`, `de`, `es`, `ky`, `uz`, `kk`

## Структура проекта

```txt
src/
  components/          Astro-компоненты интерфейса
  data/centers_formatted/
                       MDX-файлы страниц центров
  data/worldCountries.ts
                       локализованные названия стран и флаги
  i18n/locales/        словари переводов
  layouts/             лейауты страниц
  pages/               страницы и маршруты Astro
  styles/tailwind.css  theme-токены Tailwind v4 и глобальные утилиты
```

## Запуск локально

Установить зависимости:

```bash
bun install
```

Запустить dev-сервер:

```bash
bun run dev
```

Собрать статический сайт:

```bash
bun run build
```

Посмотреть production-сборку:

```bash
bun run preview
```

## Как внести вклад

Делайте изменения небольшими и удобными для ревью.

1. Откройте issue или понятно опишите изменение в pull request.
2. Создайте отдельную ветку от актуальной основной ветки.
3. Держите pull request в рамках одной темы.
4. Не форматируйте и не меняйте unrelated-файлы.
5. Для фактических изменений добавляйте источники.
6. По возможности запускайте релевантную локальную проверку.

Хороший pull request обычно содержит:

- что добавлено или исправлено;
- ссылки на источники данных;
- скриншоты для UI-изменений;
- пометки о неполных или спорных данных.

## Как добавить или обновить центр

Данные центров находятся в `src/data/centers_formatted`.

Добавьте новый `.mdx` файл или исправьте существующий. Используйте такую структуру frontmatter:

```mdx
---
title: Название центра
pubDate: '2025-02-05T00:00:00.000Z'
type: Зарубежный
category: Татарский
source: https://example.com/
summary: Короткое нейтральное описание центра
location:
  country: Страна
  city: Город
  region: Регион
---
```

Обязательное поле:

- `title`

Допустимые значения `type`:

- `Регион РФ`
- `Зарубежный`
- `Онлайн`

Допустимые значения `category`:

- `Татарский`
- `Татаро-Башкирский`
- `Башкирский`
- `Крымотатарский`

Необязательные поля `location`:

- `country`
- `city`
- `region`
- `flag`

Если данных нет, лучше не добавлять поле, чем писать placeholder.

После frontmatter добавьте содержимое страницы в MDX: заголовок, контакты, ссылки, особенности, адрес или другую проверенную информацию. Текст должен быть фактическим, нейтральным и опираться на источники. Избегайте рекламных формулировок.

## Переводы

Словари переводов находятся в `src/i18n/locales`.

Если добавляете UI-текст:

1. Добавьте одинаковый ключ во все locale-файлы.
2. Используйте существующие пространства ключей: `nav.*`, `list.*`, `detail.*`, `copy.*`, `feedback.*`.
3. В компонентах используйте `useTranslations(locale)` и `t("key.name")`.
4. Не оставляйте пользовательский текст хардкодом в компонентах.

## UI-правила

У сайта спокойный редакционный визуальный язык:

- предпочитайте семантические Tailwind-токены из `src/styles/tailwind.css`;
- сохраняйте простые контентные лейауты;
- используйте существующие component-паттерны;
- не добавляйте новые цветовые палитры без явной причины;
- учитывайте light/dark-поведение при изменении глобальных токенов.

## Контент и SEO

- Сохраняйте canonical-логику и локализованные ссылки.
- Следите за осмысленной структурой заголовков.
- Не удаляйте источники и не искажайте факты.
- Не делайте широкое форматирование MDX без необходимости.
- Аккуратно работайте с метаданными и structured data.
