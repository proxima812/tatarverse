# UI/UX ревью проекта tatarverse

Дата: 2026-06-10
Объём ревью: главная страница, каталог центров (`/centers`), карточка центра, шапка и общий layout, фильтры/поиск.

---

## 1. Общее впечатление

Проект выглядит зрелым и аккуратным: тёмный/светлый режим, продуманная типографика (`text-depth-*` градиент в hero), squircle-радиусы, поддержка 4 локалей (ru/en/tt/qt), iconography от `astro-icon` — всё это даёт ощущение «качественного редакторского» сайта, а не дженерик-каталога. Структура файлов чистая (sections/features/ui/layout), что повышает поддерживаемость.

Но именно для пользователя главная боль — это **сценарий «нашёл центр»**: основной value сайта — каталог из ~N карточек, и весь UX каталога должен работать как часы. Здесь есть что улучшать.

## 2. Главная страница (`/`)

### Что хорошо
- Hero с убывающей насыщенностью (`text-depth-100…700`) — эффектно и узнаваемо.
- Marquee со словами добавляет «живой» культурный акцент.
- Понятный CTA-блок ссылок (`HomeLinks`).

### Проблемы
1. **Нет явного главного CTA**. Самое ценное действие — «открыть каталог» — спрятано в маленькой строке `text-xs text-primary/50` под hero. Это анти-паттерн: микро-текст с low-contrast используется как основная конверсия.
   - **Совет:** Сделайте под hero кнопку-CTA уровня `lg`/`xl`: «Открыть каталог · 123 центра» с иконкой. Маленькую текстовую ссылку оставьте как вторичную.
2. **`HomeLinks` фильтрует `links` хаком** `index < 2 || index >= links.length - 2` — это скрытая логика, которая ломается при добавлении ссылки. Для пользователя — непонятно, какие ссылки считаются «главными».
   - **Совет:** Завести в `data/links.ts` явный признак `featured: true`.
3. **Marquee бесконечный 70s** — приятно, но `prefers-reduced-motion` не уважается. Часть пользователей буквально страдает от такого.
   - **Совет:** `@media (prefers-reduced-motion: reduce) { .word-marquee-track { animation: none; } }`.
4. **Hero — это `<h1>` с 7 `<span>`-строчками**, но логически это одно предложение. Скринридер прочитает рваными кусками.
   - **Совет:** объедините в один `<h1>` с visually-grouped span’ами или дайте `aria-label` с целым текстом.
5. **`select-none` на hero** — пользователи не могут скопировать заголовок. Это редко полезно и часто раздражает.
6. **Низкий контраст подписи** `text-primary/50 text-xs` — почти наверняка проваливает WCAG AA на light theme.

## 3. Каталог `/centers`

Это самый ответственный экран — здесь UX оценивается строже всего.

### Что хорошо
- Squircle-карточки, единый тон, аккуратные ring/shadow.
- Поиск через Fuse.js с autocomplete + клавиатурной навигацией (стрелки, Enter, Escape, ARIA-роли). Это сделано на уровне, который большинство каталогов не достигает.
- Состояние фильтров пишется в URL (`writeStateToUrl`) — пользователь может шарить ссылки.
- Sticky-фильтры на desktop (`lg:sticky lg:top-12`).

### Серьёзные проблемы

#### 3.1. Layout fixed 400px sidebar
```
grid-cols-1 lg:grid-cols-[400px_minmax(0,1fr)] gap-12 sm:px-12
```
- 400px — это много для фильтров из чипсов в 2 колонки. На экранах 1280–1440px это съедает ~30% ширины и оставляет карточкам мало места — особенно вкупе с `gap-12` и `sm:px-12`.
- При этом `GridCards` всегда `grid-cols-1` — то есть каталог из десятков карточек идёт **одной колонкой**, хотя есть `data-view-btn`-переключатели вида (`viewBtn`), которые меняют только `dataset.view`, но CSS их не использует.
- **Совет:**
  - Сузить сайдбар: `lg:grid-cols-[280px_minmax(0,1fr)]`.
  - Реально реализовать переключение вида: `data-view="2"` → `lg:grid-cols-2`, `data-view="3"` → `xl:grid-cols-3`. Сейчас кнопки кликаются, но визуально ничего не меняется — это баг доверия.

#### 3.2. Фильтры: «Показать ещё» и видимость
- `countryVisibleLimit = 5`, `defaultVisibleLimit = 2` (типы/категории/регионы по умолчанию **2** видимых). Это слишком мало: если 6 категорий, пользователь видит 2 и кнопку «+4». Лишний клик на каждой группе.
- **Совет:** для типов/категорий поднять до 6–8. Для регионов — сделать скролл-контейнер с поиском по регионам.

#### 3.3. Нет визуального состояния «фильтры применены»
- Бэйдж количества фильтров есть только в кнопке мобильного toggle (`data-filters-badge`). На desktop пользователь не видит сводки: «Найдено 12 из 234 · 2 фильтра».
- **Совет:** добавить компактный summary над `GridCards`: `12 из 234 · Россия × Татарстан × ✕ сбросить`.

#### 3.4. `scrollIntoView` после каждого клика по чипсу
```js
scheduleApplyFilters(true)  →  scrollToCards()
```
Сейчас при каждом тыке в фильтр страница прокручивается к сетке. Если пользователь скроллит сайдбар и подбирает фильтры — каждый клик его «вышвыривает». Это анти-паттерн.
- **Совет:** скроллить только при первом применении фильтра в сессии или на mobile, где сайдбар модальный.

#### 3.5. Карточка `CardItem` — навигационная иерархия
- На карточке справа сверху лежит pill «Подробнее» (`t("card.details")`) с `ring`-стилем — он визуально перетягивает внимание с города и страны слева, хотя вся карточка уже `<a>`. Двойной CTA, конкурирующий с заголовком.
- В подвале справа — `Обновлено: июнь 2025`, тоже видный. В итоге глаз бегает: иконки → флаг/город → «Подробнее» → заголовок → саммари → тип/категория → дата. Нет чёткой Z-структуры.
- **Совет:**
  - Убрать pill «Подробнее» — вся карточка кликабельна; стрелку показывать только на hover.
  - Метаданные (тип/категория/дата) спустить в одну строку с low-contrast — это контекст, не CTA.
  - Заголовок поднять выше — он должен быть точкой входа взгляда.

#### 3.6. Поиск
- Контейнер `bg-surface/90 ... ring ... shadow-2xs ... backdrop-blur` на mobile превращается в плавающую панель — но `relative z-40` внутри `lg:static` ломает sticky-поведение. На длинной странице поиск уезжает наверх, и пользователь не может уточнить запрос, не возвращаясь.
- **Совет:** на mobile сделать поиск `sticky top-0` с тонкой тенью при скролле.
- Suggestions: `w-[135%]` на mobile — обрезается, потенциально вылазит за viewport, ломая layout.
- Нет кнопки очистки в input (`type="search"` даёт нативный «×», но он зависит от браузера и не очищает фильтры).
- **Совет:** добавить визуальный «×» внутри input + reset фильтров.

#### 3.7. `BackButton href="/"` на странице каталога
Это странно: пользователь зашёл на `/centers` напрямую — стрелка «назад» ведёт на главную, а не в историю. Текст «Back». На каталоге как такового «обратно» нет — это сайт верхнего уровня.
- **Совет:** убрать совсем со страницы каталога или заменить на хлебные крошки `Главная / Каталог`.

#### 3.8. Доступность фильтров
- Чипсы — это `<button>` с `aria-pressed`, хорошо. Но `aria-label` групп подтягивается из i18n и проверить, что переводы есть для всех локалей, — отдельная задача.
- Контраст активных чипсов (`data-[active]:bg-primary`) почти наверняка ок, но «не активных» (`bg-muted text-muted-foreground`) на тёмной теме часто проваливает 4.5:1 — проверить.

## 4. Карточка центра `/centers/[id]`

### Что хорошо
- Sticky-сайдбар с фактами справа — стандартный, понятный паттерн.
- Кнопка «Edit on GitHub» — это открытость, аудитория такое любит.
- Pagination prev/next в конце.

### Проблемы
1. **Hero отсутствует — заголовок закомментирован** (строки 160–164). На странице нет крупного `<h1>` — пользователь видит сразу навигацию и контент markdown. Это плохо и для SEO, и для иерархии.
   - **Совет:** раскомментировать или переделать в новый hero-блок. `<h1>` — обязателен.
2. **Верхняя `nav` перегружена**: «All centers», «Find a mistake?», «Edit on GitHub», `CopyPageMenu` — 4 действия в одной строке, без визуальной иерархии. На мобиле они оборачиваются и занимают 3 строки.
   - **Совет:** оставить слева только «← All centers», остальные действия собрать в overflow-меню («⋯») или вынести в подвал страницы.
3. **`pageDescription` = `${pageDescription}`** — если `summary` пусто, в meta-description попадёт строка `"undefined"`. SEO-баг.
   - **Совет:** `description={pageDescription}` без шаблонной строки; в `Layout` падать в дефолт.
4. **`facts` имеет timezone `Asia/Almaty`** — почему дата обновления карточки в Алматы? Для российских/европейских пользователей это сбивает с толку и может сместить дату на день.
   - **Совет:** UTC или локаль пользователя.
5. **`sourceLabel`** показывает только hostname — это аккуратно, но без favicon/иконки выглядит как «голый текст». Маленький `🌐` или favicon усилит распознавание.
6. **`TranslationFeedbackCard`** в самом низу — далеко от контента. Если пользователь нашёл ошибку перевода в первой трети текста, скролл до фидбэка длинный.
   - **Совет:** добавить плавающую кнопку «Сообщить об ошибке» или анкор `#errors` в sidebar.

## 5. Header и навигация

### Проблемы
1. `ThemeToggle hidden off` — переключатель темы **скрыт**. При этом проект полностью поддерживает тёмную/светлую тему. Пользователь не может выбрать.
   - **Совет:** включить toggle. Это бесплатная фича, которой пользователи ожидают.
2. Header не sticky. На длинной странице каталога логотип и языковой переключатель уезжают — приходится скроллить наверх, чтобы сменить язык.
   - **Совет:** `sticky top-0 z-30 backdrop-blur` + лёгкая тень при скролле.
3. `flex-wrap` в header — на маленьких ширинах меню может перейти на новую строку, ломая визуальный ритм. Лучше hamburger.

## 6. Сквозные UX-замечания

1. **404-страница** — есть файл, но если в ней нет ссылки на каталог и поиска — её надо доработать. Каждая 404 — это потерянный пользователь, которого можно вернуть в каталог.
2. **Loading states**. Поскольку каталог рендерится статикой (Astro), это не критично, но autocomplete-suggestions не показывают «нет подсказок» — список просто прячется. Дайте feedback.
3. **Empty states**. `NoResults` есть, но я бы добавил туда кнопку «Сбросить все фильтры» прямо рядом с сообщением.
4. **Микрокопия**. «Back», «All centers», «Find a mistake?», «Edit on GitHub» — это языковой микс (в ru-локали должно быть «Все центры»). Проверьте, что ключи переводов реально применяются (в JSX стоят `label="Back"` как fallback — если ключ потерян, увидите английский).
5. **Анимации**. Hover-переходы `transition duration-300` на карточках — ок, но 300ms на цвет/тень — на грани заметности. 150–200ms живее.
6. **Touch targets**. Чипсы фильтров `py-1.5 px-2.5 text-sm` ~30px высотой. WCAG 2.5.5 рекомендует 44×44px. На mobile палец может промахиваться.
7. **Focus styles**. На карточке `focus-visible:outline-2 outline-primary` — ок. На чипсах фильтра явного focus-стиля не вижу — там нужен `focus-visible:ring-2`.
8. **SEO keywords-спам**. В `index.astro` и `centers/index.astro` массивы `keywords` забиты до 14 фраз, включая динамические (`${cards.length} карточек ТБК`). Поисковики такое игнорируют или штрафуют. Достаточно 4–6.
9. **`select-none` на hero** уже упомянул — это вредит UX без выгоды.

## Резюме

Сайт сделан с любовью и хорошим вкусом — типографика, иконография, i18n и поиск выше среднего. Главные потери UX — в (1) скрытом основном CTA на главной, (2) недоиспользованной сетке каталога (всегда одна колонка + переключатель-плацебо), (3) скрытом ThemeToggle, (4) отсутствии `<h1>` на карточке центра и (5) агрессивном `scrollIntoView` фильтров. Эти 5 правок дают самый большой прирост ощущения «всё работает как надо».
