# UI/UX — каталог tatarverse

Единый рабочий документ по UI/UX. Объединяет прежние `plan-ui-ux.md` и `UI-UX-REVIEW.md`
(оба были архивными заметками). Здесь — актуальный контекст, что уже сделано и что ещё стоит
проверить. Перед реализацией сверяйте пункты с текущим кодом, а не переносите механически.

Дата последней сверки: 2026-06-23.

## Контекст продукта

- Активные публичные локали: `ru` и `en`. Языки `tt`, `qt`, `ba` публичными не делать без отдельного решения.
- Русские карточки центров — исходные, лежат в `src/data/centers_formatted` (365 шт.).
- Английские карточки — в `src/data/centers_i18n/en`.
- UI-переводы — `src/i18n/locales/ru.ts` и `src/i18n/locales/en.ts`.
- Основной сценарий: быстро найти нужный центр и перейти к проверяемой информации.

## Архитектурные ограничения

- Сохранять практичную, монохромную, каталоговую подачу. Не превращать в маркетинговый лендинг.
- Не копировать тему/цвета/подачу `dualmark.dev`.
- Не добавлять новую палитру, декоративные градиенты, лишние анимации.
- Не вводить новые поля данных (например `group`), если их нет в `src/content.config.ts`.
- Каталог `/centers` рендерит все карточки на сервере (SEO), фильтрация — клиентский
  императивный скрипт в [src/scripts/centersToolbar.ts](src/scripts/centersToolbar.ts).
  Реактивный фреймворк здесь не нужен: при 365 SSR-карточках ваниль производительнее
  (карточки не гидрируются). Подробности решения — в истории обсуждения.

## Сделано

- Логика тулбара вынесена из `Toolbar.astro` в отдельный TS-модуль
  [src/scripts/centersToolbar.ts](src/scripts/centersToolbar.ts) без смены парадигмы.
- `scrollIntoView` при фильтрации: на десктопе (`min-width: 1024px`) автоскролл отключён —
  панель фильтров сбоку, сетка и так в зоне видимости; скролл остался только на мобиле.
- На мобиле скролл уважает `prefers-reduced-motion` (`auto` вместо `smooth`).
- Убраны нерабочие кнопки переключения grid-view (мёртвый код + отсутствие разметки).

## Что проверять при будущих правках

1. Главная страница должна ясно вести в каталог (явный CTA).
2. Каталог `/centers`: поиск, фильтры, клавиатурная навигация, понятные empty states,
   читаемая сводка применённых фильтров.
3. Карточка `/centers/[id]`: корректный `<h1>`, source link, metadata, локализованные действия.
4. Английские routes сохраняют фактический смысл русских source-записей.
5. Контраст, focus-states и touch targets — проверять на mobile и desktop (375 / 768 / 1280).
6. Локализация всего видимого текста без англоязычного fallback в русской версии
   (в `index.astro` keywords захардкожены на ru — проверять `/en`).
7. Meta-description и canonical на страницах центров.

## Возможные улучшения (с отдельным решением)

Из архива плана — можно использовать, но не обязательно и только по реальным данным:

- Компактная секция статистики на главной из существующих полей.
- Быстрый поиск с главной, ведущий в `/centers`.
- Географическая сводка по странам/городам, если данные готовы.
- Contributor CTA на `CONTRIBUTING.ru.md` / `CONTRIBUTING.en.md`.

Не делать без отдельного решения: длинный лендинг ради ритма, спотлайт/ротация центра без
понятного source-backed правила выбора, моноширинные code-block мотивы как декор.

## Рабочий порядок UI-правок

1. Начинать с малого, точечными изменениями.
2. Проверять адаптивность на 375 / 768 / 1280.
3. Проверять обе локали `ru` и `en`.
4. Запускать targeted Astro check или full build, если менялись маршруты, config или shared data logic.
