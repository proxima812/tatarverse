# План UI/UX: перенос структуры dualmark.dev на tatarverse

## Источник вдохновения

`dualmark.dev` — продуктовый лендинг с очень чистой секционной структурой, одним ярким акцентом (лаймовый зелёный) на тёмном фоне, крупной типографикой и моноширинными блоками как декоративным элементом. Цель — взять *композицию* (ритм секций, иерархию, паттерны блоков), а не цветовую тему 1-в-1.

---

## 1. Разбор структуры dualmark.dev

| # | Секция | Роль | Визуальный паттерн |
|---|--------|------|--------------------|
| 1 | Hero | Заявление миссии + первичный CTA | Крупный заголовок с акцентным словом, подзаголовок, ярко-зелёная кнопка, превью-карточка ниже |
| 2 | «Score your site» | Интерактивный инструмент | Центр, инпут URL + кнопка, минимум текста |
| 3 | «Two formats» | Демонстрация ценности | Две колонки с код-блоками рядом, моноширинный |
| 4 | «5× / 60d / 0» | Социальное доказательство через числа | Три крупных числа в ряд, короткие подписи |
| 5 | «One command» | Пример использования | Терминал-блок по центру, тёмная карточка с кодом |
| 6 | «Drop into your stack» | Совместимость / экосистема | Сетка логотипов 3×N с короткими подписями |
| 7 | «Ship it in 30 seconds» | Финальный CTA | Повтор героя в компактной форме, одна кнопка |
| 8 | Футер | Навигация и юр. информация | 4–5 колонок ссылок |

**Ключевые принципы дизайна:**
- Один акцентный цвет, применяется строго (CTA + одно слово в заголовке)
- Тёмный фон → светлый текст, контраст высокий
- Каждая секция дышит — большие вертикальные отступы (~120–160px между блоками)
- Центрирование, узкий контент-контейнер (~720–880px)
- Моноширинные блоки как ритм-элемент, не как контент сам по себе
- Минимум декора, никаких градиентов и иллюстраций

## 2. Текущее состояние tatarverse

`src/pages/index.astro` сейчас:
- `HomeHeroTitle` — герой с заголовком
- `HomeLinks` — ссылки навигации
- Параграф-описание + кнопка «Открыть каталог»
- `WordMarquee` — бегущая строка слов

Стек: Astro + Tailwind, светлая/тёмная тема через CSS-переменные, компоненты в `src/components/sections/home/`.

**Проблемы текущей главной:**
- Нет ритма — всё в одном вертикальном стеке без выраженных секций
- Каталог (главная ценность продукта) упоминается, но не показан
- Нет статистики, нет «зачем» проект, нет точки входа для разных аудиторий

## 3. Маппинг секций: dualmark → tatarverse

| dualmark.dev | tatarverse | Содержание |
|---|---|---|
| Hero «AEO infrastructure…» | **Hero «Найди свой центр»** | Крупный заголовок, акцентное слово («центр» или «свой»), подзаголовок про миссию, CTA «Открыть каталог · {N} центров» |
| Score your site (инпут) | **Быстрый поиск** | Поле «Город или страна» → переход в `/centers?q=...` |
| Two formats (две колонки кода) | **Три народа** | Три колонки: татары / башкиры / крымские татары — короткое описание + кол-во центров каждой группы |
| Большие числа 5× / 60d / 0 | **Каталог в цифрах** | `{centers}` центров · `{countries}` стран · `{cities}` городов |
| One command (терминал) | **Спотлайт месяца** | Карточка одного центра/события — фото, название, город, короткое описание, ссылка |
| Drop into your stack (логотипы) | **География** | Сетка флагов/названий стран, где есть центры (с числом) |
| Ship it in 30 seconds | **Добавь свой центр** | Финальный CTA — короткий блок «Не нашли? Добавьте» с одной кнопкой |
| Footer | **Футер** | Языки (ru/tt/qt/ba), разделы, источники, политика |

## 4. Детальная спецификация секций

### 4.1 Hero
- **Контейнер:** centered, max-w ~880px, padding-top ~120px, padding-bottom ~80px
- **Заголовок:** h1, 56–72px, line-height 1.05, weight 600
  - Текст: «Татарские, башкирские и крымскотатарские **центры** по всему миру»
  - Акцентное слово — primary цвет
- **Подзаголовок:** 18–20px, muted-foreground, max-w ~640px, line-height 1.5
  - Текст: «Каталог сообществ и инициатив для встреч, языка, культуры и общих ценностей.»
- **CTA:** primary button, «Открыть каталог · {N} центров», corner-squircle
- **Вторичная ссылка:** «Добавить свой центр» текстом справа от кнопки или ниже
- **Превью-карточка ниже** (опционально, на v2): мини-карусель 3 случайных центров

### 4.2 Быстрый поиск
- **Контейнер:** centered, max-w ~640px
- **Заголовок:** h2, 32–40px, «Найди ближайший»
- **Подзаголовок:** короткая строка, «Город, страна или название»
- **Форма:** один инпут + кнопка, как у dualmark — submit ведёт на `/centers?q=...`
- На v1 — простое поле без автокомплита; на v2 — клиентский фильтр поверх предзагруженного списка

### 4.3 Три народа (две/три колонки)
- **Контейнер:** max-w ~1080px
- **Заголовок:** h2, «Один каталог. Три народа.»
- **Сетка:** 3 колонки на десктопе, 1 на мобайле, gap ~24px
- **Каждая карточка:**
  - Название группы (татары / башкиры / крымские татары)
  - 1–2 строки контекста
  - Большое число «{N} центров»
  - Ссылка «Смотреть →» с фильтром по группе
- Карточки на subtle border, без фона, hover — лёгкая подсветка border

### 4.4 Каталог в цифрах
- **Контейнер:** centered, max-w ~960px
- **Заголовок:** h2, «Каталог в цифрах»
- **Сетка 3 колонки:** число (56–72px, bold) сверху + подпись (muted) снизу
  - `{centers}` — центров
  - `{countries}` — стран
  - `{cities}` — городов
- Над числами — короткая фраза-вступление одной строкой

### 4.5 Спотлайт
- **Контейнер:** centered, max-w ~720px
- **Заголовок:** h2, «Центр месяца» или «Сегодня в каталоге»
- **Карточка:** темнее фон, padding ~32px, border-radius corner-squircle
  - Название, город, страна
  - 2–3 строки описания
  - Ссылка на страницу центра
- На v1 — статически выбранный или первый из коллекции; на v2 — ротация

### 4.6 География
- **Контейнер:** max-w ~1080px
- **Заголовок:** h2, «Где мы есть»
- **Подзаголовок:** «Центры в {N} странах»
- **Сетка:** 4–6 колонок флагов/стран, каждый элемент — название страны + кол-во центров, кликабельный → фильтр
- Стиль: subtle border-box, без фона, как «Drop into your stack»

### 4.7 Финальный CTA
- **Контейнер:** centered, max-w ~640px
- **Заголовок:** h2, «Не нашли свой центр?»
- **Подзаголовок:** «Добавьте — мы проверим и опубликуем»
- **CTA:** primary button «Добавить центр» → ссылка на форму/email/PR-инструкцию

### 4.8 Футер
- **Контейнер:** max-w ~1080px, 4 колонки на десктопе
  - **Колонка 1:** логотип tatarverse + 1 строка миссии
  - **Колонка 2:** Разделы — Каталог, Источники, Переводы, Политика
  - **Колонка 3:** Языки — ru / tt / qt / ba
  - **Колонка 4:** Контакт / GitHub / соцсети
- Нижняя строка: копирайт, версия

## 5. Дизайн-система: что использовать и что добавить

### Уже есть в проекте
- CSS-переменные `--primary`, `--muted-foreground`, `--background`, `--foreground`
- `corner-squircle` утилита для скруглений
- Tailwind с typography-классами

### Что добавить или согласовать
- **Spacing token:** `section-y` = `py-20 sm:py-28 lg:py-32` для согласованных вертикальных отступов
- **Container token:** три ширины — `max-w-3xl` (текст), `max-w-5xl` (статы, сетки), `max-w-6xl` (география, футер)
- **Display heading:** новый класс/компонент `<DisplayHeading>` с размерами 56/72/96 для героев секций
- **Stat block:** компонент `<StatBlock value="..." label="..." />` для секции цифр
- **Section heading:** `<SectionHeading title="..." subtitle="..." />` для согласованности

### Акцент
- НЕ копировать лаймовый dualmark. Использовать текущий `--primary` проекта (или скорректировать на татарский зелёный/изумруд один раз, глобально)
- Применять строго: одна CTA-кнопка в секции, одно акцентное слово в h1, нигде больше

| мобайл (<640px) | 36–44px | py-16 | 1 |

Превью-карточка героя и спотлайт сжимаются в одну колонку на мобайле без потери смысла.

## 7. Контент и данные

Что нужно вытащить из коллекций (`src/content.config.ts`):

- `centers.length` — для CTA героя
- Группировка по `group` (татары/башкиры/крымские татары) → подсчёт для секции «Три народа»
- Уникальные `country` → для секции «География»
- Уникальные `city` → для счётчика «Каталог в цифрах»
- Один центр для спотлайта (на v1 — по `slug` в конфиге, на v2 — по дате/ротации)

Если поля `group` / `country` / `city` в схеме отсутствуют — добавить в `src/content.config.ts` и заполнить миграцией.

## 8. План реализации по фазам

### Фаза 1: каркас (1 итерация)
- Создать компоненты-обёртки: `Section.astro`, `SectionHeading.astro`, `StatBlock.astro`
- Переписать `index.astro` под новый секционный каркас
- Hero + быстрый поиск (без логики) + цифры + финальный CTA
- Один цвет акцента согласован

### Фаза 2: контент (1 итерация)
- Секция «Три народа» — карточки с реальными числами из коллекции
- Секция «География» — сетка стран с числами
- Спотлайт центра — статический выбор

### Фаза 3: интерактив (1 итерация)
- Быстрый поиск с переходом на `/centers?q=...` (страница каталога должна принимать query)
- Hover-состояния, focus-states, переходы
- Тестирование тёмной/светлой темы

### Фаза 4: финиш (опционально)
- Превью-карусель в герое
- Ротация спотлайта
- Микроанимации появления при скролле

## 9. Что НЕ делать

- Не копировать лаймовый зелёный dualmark — это их фирменный цвет, у нас своя палитра
- Не делать длинный лендинг ради лендинга — каталог должен быть в один клик из героя
- Не вводить декоративные градиенты/блюры/иллюстрации — стиль dualmark держится на пустоте, не на украшениях
- Не добавлять моноширинные код-блоки буквально (они уместны у dualmark как продукта, у нас будут чужеродны) — заменить на текстовые карточки с тем же ритмом
- Не плодить акцентные цвета — один primary везде

## 10. Критерии готовности

- [ ] На главной видны 7 секций с одинаковым вертикальным ритмом
- [ ] Hero ведёт в каталог одним кликом, число центров корректное
- [ ] Все числа в секциях статистики берутся из реальных данных коллекции
- [ ] Адаптивность: проверено на 375 / 768 / 1280 / 1920
- [ ] Тёмная и светлая темы — обе выглядят чисто
- [ ] Lighthouse: contrast ≥ AA, нет CLS, LCP < 2s
- [ ] Контент на ru готов; tt/qt/ba — заглушки помечены TODO
