# UI/UX Plan Archive

Исходный план описывал перенос секционной структуры `dualmark.dev` на `tatarverse`. Документ сохранён как архив идей, а не как обязательная спецификация.

## Актуальные ограничения

- Не копировать визуальную тему, цвета или продуктовую подачу `dualmark.dev`.
- Не превращать каталог в маркетинговый лендинг.
- Не добавлять новую палитру, декоративные градиенты или лишние анимации.
- Сохранять текущую практичную, монохромную, каталоговую подачу.
- Учитывать только активные локали `ru` и `en`.

## Что можно использовать из плана

- Более явный вход в каталог с главной страницы.
- Компактную секцию статистики по реальным данным.
- Быстрый поиск, ведущий в `/centers`.
- Географическую сводку по странам и городам, если данные готовы.
- Финальный contributor CTA, ведущий к `CONTRIBUTING.ru.md` или `CONTRIBUTING.en.md`.

## Что не использовать без отдельного решения

- Языки `tt`, `qt`, `ba` как активные публичные локали.
- Новые поля данных вроде `group`, если их нет в `src/content.config.ts`.
- Длинный лендинг ради декоративного ритма.
- Спотлайт или ротацию центра без понятного source-backed правила выбора.
- Моноширинные code-block мотивы как декоративный элемент.

## Реализационный ориентир

Если план будет возвращён в работу, начинать нужно с малого:

1. Обновить главный CTA на главной.
2. Добавить статистику только из существующих полей.
3. Проверить адаптивность на 375 / 768 / 1280.
4. Проверить локали `ru` и `en`.
5. Запустить targeted Astro check или full build только если менялись маршруты, config или shared data logic.
